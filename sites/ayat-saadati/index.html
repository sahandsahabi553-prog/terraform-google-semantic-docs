import { useState, useEffect } from 'react';

const useFetchData = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true);
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error(`HTTP error! status: ${response.status}`);
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]); // Re-run effect if URL changes

  return { data, loading, error };
};

Performance as a Feature: Optimizing for the Human Experience

In our instant-gratification world, web performance isn't just a nice-to-have; it's a non-negotiable requirement. Studies consistently show that slow-loading websites lead to higher bounce rates, reduced conversions, and a generally frustrated user base. Google's own research indicates that as page load time goes from 1 second to 3 seconds, the probability of bounce increases by 32%. Ayat Saadati understands this implicitly, treating performance not as an afterthought but as a fundamental feature of any successful application.

Their work often delves into practical strategies for optimizing web performance, covering a broad spectrum of techniques:

  • Code Splitting and Lazy Loading: Breaking down large JavaScript bundles into smaller, on-demand chunks, ensuring users only download the code they need for the current view. This dramatically improves initial load times.
  • Image Optimization: Leveraging modern image formats (like WebP), responsive images, and lazy loading images below the fold to reduce visual payload without sacrificing quality.
  • Server-Side Rendering (SSR) and Static Site Generation (SSG): For content-heavy applications, rendering pages on the server or pre-building them at compile time can significantly improve perceived performance and SEO.
  • Efficient Asset Delivery: Utilizing CDNs, minification, compression (Gzip, Brotli), and browser caching to deliver assets as quickly as possible.
  • Critical CSS: Inlining only the CSS required for the initial viewport to render content above the fold almost instantly, reducing render-blocking CSS.

I recall a project where we shaved nearly two seconds off the initial page load simply by implementing a few of these techniques. The impact on user engagement was almost immediate and measurable. Ayat's discussions often highlight these tangible benefits, emphasizing that optimizing performance is a direct investment in user experience and business metrics. They're not just quoting best practices; they're explaining why they matter and how to implement them effectively. It's this pragmatic, results-oriented perspective that makes their guidance so valuable.
